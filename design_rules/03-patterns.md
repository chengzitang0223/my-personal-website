# 03-Patterns: 布局与交互模式

## 1. 核心布局模式
* **Hero Section (首屏大图)**：顶部用于展示核心卖点（如场馆环境）。居中对齐，大标题 + 副标题 + 胶囊按钮。
* **Bento Box (便当盒网格)**：用于管理后台或数据看板（Dashboard）。将不同指标（如今日上课人数、待处理表单）放在大小不一、圆角极大的区块中，形成错落有致的网格布局（使用 CSS Grid `grid-cols-2 md:grid-cols-4 gap-4`）。

## 2. 交互与动画 (Motion)
* **平滑性**：所有状态切换（Hover、展开/收起）必须带有平滑过渡 `transition-all duration-300 ease-in-out`。
* **滚动吸顶 (Sticky Header)**：页面向下滚动时，分类导航栏需平滑吸附在顶部，并开启毛玻璃背景。

## 3. 业务场景特定模式
* **选课流**：不要把所有课程挤在一个长列表。使用横向滚动卡片（Carousel）展示推荐课程，下方用极简列表展示全部课程。
* **AI Agent 操作区**：如果系统内嵌 AI 助手，对话框或操作浮层应采用带有阴影和毛玻璃效果的悬浮面板，仿佛漂浮在系统内容之上。
